# 🚕 Uber Fare Prediction – Production ML Pipeline

End-to-end production-ready machine learning pipeline for predicting Uber fares in New York City.  
The model (CatBoost + sklearn Pipeline) is served via a Flask web application with an interactive Leaflet map.

## 🗺️ How It Works

1. User **clicks pickup** location on the NYC map (Leaflet + OpenStreetMap)  
2. User **clicks dropoff** location  
3. JavaScript stores lat/lon in hidden form fields  
4. User fills in: datetime, passengers, weather, car condition, traffic  
5. Flask receives **raw inputs only** – no engineered features  
6. `feature_engineering.create_features()` computes all 22 features  
7. The loaded CatBoost pipeline predicts the fare  
8. Result is shown on a confirmation page with a route map  

> ✅ The user **never** manually enters: distance, bearing, airport distances, cyclic features.

## 🔬 Feature Engineering

All preprocessing lives in `feature_engineering.py` and is shared between training and inference:

| Feature | Description |
|---|---|
| `distance` | Haversine great-circle distance (km) |
| `bearing` | Direction angle pickup→dropoff (radians) |
| `jfk_dist` | Distance to JFK Airport (km) |
| `lga_dist` | Distance to LaGuardia Airport (km) |
| `nyc_dist` | Distance to NYC city center (km) |
| `hour_sin/cos` | Cyclic encoding of pickup hour |
| `weekday_sin/cos` | Cyclic encoding of day of week |
| `month_sin/cos` | Cyclic encoding of month |
| `rush_hour` | 1 if 7–9AM or 5–10PM |
| `is_weekend` | 1 if Saturday or Sunday |
| `is_night` | 1 if 11PM–5AM |
| `Car Condition` | Ordinal: Bad=0, Good=1, Very Good=2, Excellent=3 |
| `Traffic Condition` | Ordinal: Flow=0, Dense=1, Congested=2 |
| `Weather` | OneHotEncoded (sunny/cloudy/windy/stormy) |

## 🛡️ Training/Serving Skew Prevention

The golden rule: **one function, two places**.

```python
# Training:
from feature_engineering import create_features
X_train = create_features(df_train)

# Inference:
from feature_engineering import build_inference_dataframe
X = build_inference_dataframe(pickup_lon, pickup_lat, ...)
```

Both code paths produce identical DataFrames – same columns, order, and dtypes.
